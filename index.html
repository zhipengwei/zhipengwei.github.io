    • Research Experiences

    01/2014 - Present

  • Evaluation of cache set balance on Nehalem and Sandy Bridge processors.

    Based on HMTT, I develop the function that could record the reference time of all cache sets during program execution; Then I evaluated the cache set balance on Nehalem and found that different cache sets experienced very different levels of demand, and even an exact set experienced different level of command during program execution. Unawareness of the hash function on Sandy Bridge processors by which all physical addresses are distributed across all cache slices prevented me from getting set address directly from the physical address. So I use statistical methods to crack the hash function.

    • 03/2013 - 10/2013

  • KSM Optimization based on HMTT..

    Based on the memory reference traces collected using HMTT, all physical pages were classified into different categories based on the reference characteristics of the pages. Unlike the current implementation that only two global red black trees are constructed in KSM, one tree is constructed for the pages in each category. In order to find chance of page merging, the candidate page only needs to be compared with the pages that share the same reference characteristics rather than compared with all physical pages. In this way, futile page comparisons are reduced by 68.5%.

    • 03/2014 - 06/2014

  • Implementation of the improved version of HMTT..

    I implemented the improved version that can collect traces from two memory channels. The main duties were to implement the mechanism that synchronized the time stamps of the memory reference traces of two memory channels and to implement the function that encapsulated data into transaction layer packets.
