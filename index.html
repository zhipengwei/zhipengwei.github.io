    • Research Experiences

    01/2014 - Present

  • Evaluation of cache set balance on Nehalem and Sandy Bridge processors.

    Firstly, I developed the function based on HMTT that could record the reference time of all cache sets during the whole execution period of one program; Evaluated the cache set balance on Nehalem and found that different cache sets can experience very different levels of demand, and even a exact set can experience different level of command during the period of execution; As a comparison, I want to evaluate cache set balance on Sandy Bridge processor. Because of the hash function by which all physical addresses are distributed across all cache slices, I am not able to get set addresses directly from the physical addresses. So I used statistical methods to try to crack the hash function.

    • 03/2013 - 10/2013

  • KSM Optimization based on HMTT..

    Based on the memory reference traces collected using HMTT, a few characteristics can be extracted. Furthermore, all the physical pages can be classified into different categories based on these characteristics. And unlike the current implementation that only two global red black trees are constructed in KSM, one tree is contructed to hold the pages in each catogary. As a result, the candidate page only need to be compared with the page category that shares the same characteristics with itself rather than compared with all physical pages. As a result, futile page comparisions are reduced by 68.5%.

    • 03/2014 - 06/2014

  • Implementation of the improved version of HMTT..

    HMTT is a HMTT: A Hybrid Memory Trace Toolkit for the Real Systems to collecting memory reference traces. I implement the improved version that can collect traces from two memory channels. My major work is that I implemented the mechanism that synchronizes the time stamps of the memory reference traces from two memory channels. Besides, I implement the toolkit that decodes the trace and the driver that serves as the control interface of the tool;
